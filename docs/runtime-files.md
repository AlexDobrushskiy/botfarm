# Runtime Files

Botfarm stores all runtime data under `~/.botfarm/` by default. This document describes the full directory layout.

## Directory Tree

```
~/.botfarm/
├── config.yaml                          # Main configuration
├── .env                                 # Environment variables (auto-loaded)
├── botfarm.db                           # SQLite database (all persistent state)
├── logs/
│   ├── supervisor.log                   # Supervisor log (rotated)
│   ├── supervisor.log.1 … .5           # Rotated backups
│   └── <TICKET-ID>/                     # Per-ticket log directory
│       ├── worker.log                   # Worker process log (rotated)
│       └── <stage>[-iter<N>]-<timestamp>.log  # Claude subprocess output
└── slots/
    └── <project>-<slot_id>/             # Temporary sandboxed DB (auto-cleaned)
        └── botfarm.db
```

## Files

### `config.yaml`

Main YAML configuration. Generated by `botfarm init`. Can be overridden with `--config <path>` on any CLI command. See [configuration.md](configuration.md) for full reference.

### `.env`

Environment variables loaded automatically via `python-dotenv` on every CLI invocation. Typical contents:

```
LINEAR_API_KEY=lin_api_xxxxxxxxxxxxxxxxxxxx
```

### `botfarm.db`

SQLite database (WAL mode) containing all persistent state: slot records, task history, stage runs, usage snapshots, and task events. Created automatically on first run. Path is configurable via `database.path` in config.

## Logs

### `logs/supervisor.log`

Main supervisor process log. Uses `RotatingFileHandler` — rotates when it reaches `max_bytes` (default 10 MB), keeping up to `backup_count` (default 5) old files.

### `logs/<TICKET-ID>/worker.log`

Per-ticket worker process log. Same rotation settings as the supervisor log.

### `logs/<TICKET-ID>/<stage>[-iter<N>]-<timestamp>.log`

Raw output from each Claude subprocess invocation. Filename format:

- `implement-20250225-143000.log` — first implement run
- `review-20250225-144500.log` — first review
- `fix-iter2-20250225-150000.log` — second fix iteration

These are the most useful files for debugging what Claude actually did during a stage.

### Log retention

Old per-ticket log directories are cleaned up hourly. Default retention is 30 days (`logging.ticket_log_retention_days` in config). Directories for active tickets are never removed.

### Log configuration

All log settings live under the `logging` key in `config.yaml`:

```yaml
logging:
  max_bytes: 10485760           # 10 MB default
  backup_count: 5               # rotated files to keep
  ticket_log_retention_days: 30 # auto-cleanup threshold
```

The `--log-dir` flag on `botfarm run` overrides the default `~/.botfarm/logs/` location.

## Temporary Slot Directories

### `slots/<project>-<slot_id>/botfarm.db`

When a worker is dispatched, the supervisor copies the main database into a per-slot sandbox at `~/.botfarm/slots/<project>-<slot_id>/botfarm.db`. This sandboxed copy is passed to the Claude subprocess via the `BOTFARM_DB_PATH` environment variable so the agent can query task context without touching the production database.

These directories are automatically removed (`shutil.rmtree`) when the slot finishes — whether it succeeds, fails, or times out. They should not accumulate under normal operation.
